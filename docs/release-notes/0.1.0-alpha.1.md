# pyfallow 0.1.0-alpha.1

`pyfallow` is an early Python-first static codebase intelligence tool for agents and reviewers.

It exists to give coding agents and human reviewers a deterministic, machine-readable view of Python codebase health without importing or executing the project under analysis.

## Main Capabilities

- source discovery and module resolution
- AST indexing and import graph output
- circular dependency detection
- likely dead module and unused top-level symbol detection
- dependency declaration analysis
- duplicate code detection
- cyclomatic and cognitive complexity approximations
- architecture boundary rules
- suppressions and stale suppression reporting
- JSON, text, SARIF, graph, baseline, and agent-context output

## Recommended Use

- Use `agent-context` before broad code edits.
- Use JSON output for automation.
- Use SARIF for code scanning consumers.
- Use baselines to adopt pyfallow in CI without blocking on existing debt.
- Do not auto-delete low-confidence dead code.

## Install

```bash
python -m pip install -e ".[dev]"
```

First command:

```bash
python -m pyfallow analyze --root examples/demo_project --format text
```

## Limitations

Python is dynamic. Findings are evidence-backed signals, not runtime proof. Dynamic imports, monkey patching, reflection, plugin loading, framework magic, runtime path mutation, and public APIs consumed outside the repository can all affect accuracy.

## Feedback

False positives are valuable. Please open a false-positive issue with the rule, finding excerpt, code pattern, framework/library involved, and the confidence adjustment you expected.
